# Тестовое задание PHP Developer

Написать на базе фреймворка Yii2 веб-приложение для парсинга и анализа логов веб-сервера nginx.

Формат логов
127.0.0.1 - - [21/Mar/2019:00:20:06 +0300] "GET /favicon/favicon-32.png HTTP/1.1" 200 1306 "/catalog/catalog_edit?id=4" "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/73.0.3683.75 Safari/537.36"

## Что нужно парсить

- IP адрес
- Дата/время запроса
- URL и HTTP-статус
- User-Agent, из которого дополнительно извлекается:
    - Операционная система
    - Архитектура (x86 / x64 / unknown)
    - Браузер
    - Признак бота (Googlebot, YandexBot и т.д.)

## Куда сохранять данные

В БД MySQL. Объекты БД создавать через миграции.

## Как загружать данные

Загрузка файла через веб-форму работает асинхронно: задание на импорт помещается в очередь на базе Redis (LIST). Отдельная консольная команда читает очередь и сохраняет записи в БД батчами. Повторная загрузка того же файла не должна создавать дубли. Прогресс импорта (сколько строк обработано) хранится в Redis и отображается на странице без перезагрузки.

## Как отображать данные

Данные отображаются на одной странице в виде графиков и таблицы.

Графики
- По оси X - даты, по оси Y - число запросов (люди и боты отдельными рядами)
- По оси X - даты, по оси Y - доля (% от числа запросов) для трёх самых популярных браузеров

Колонки таблицы
- Дата
- Число запросов
- Самый популярный URL
- Самый популярный браузер

## Фильтры

Данные для графиков и таблицы должны фильтроваться по:
- Дата (диапазон от и до, не более 1 года)
- ОС
- Архитектура
- Боты: все / только люди / только боты

## Кэширование

Результаты агрегирующих запросов кэшировать в Redis с TTL 5 минут. Ключ кэша зависит от параметров фильтра.

## Сортировки

Должна работать сортировка по каждой из колонок таблицы.

## Рекомендации к выполнению задания

- Использовать шаблон basic
- Всё окружение поднимается командой docker-compose up (PHP-FPM, Nginx, MySQL, Redis)
- Объекты БД создавать через миграции
- Написать unit-тесты для класса парсинга строки лога (PHPUnit)
- В процессе работы использовать Git
- Готовое задание выложить на GitHub или Bitbucket

# Nginx Log Parser (Yii2)

Веб-приложение на Yii2 (шаблон `basic`) для парсинга и анализа access-логов nginx.
Файл загружается через веб-форму, задание на импорт ставится в очередь на базе Redis,
консольный воркер построчно парсит лог и батчами сохраняет записи в MySQL без дублей.
Результаты показываются на одной странице: два графика (Chart.js) и сортируемая таблица
с фильтрами; агрегаты кэшируются в Redis на 5 минут.

## Стек

PHP 8.2 (FPM) / Yii2 basic / MySQL 8 / Redis 7 / Nginx 1.27 / Chart.js / PHPUnit.

Структура: код приложения лежит в каталоге `app/` (стандартный шаблон Yii2 basic),
инфраструктура (`docker-compose.yml`, `docker/`) - в корне репозитория.

## Требования

- Docker и Docker Compose (плагин `docker compose`).
- Свободные хост-порты (можно переопределить в `.env`): `8080` (nginx), `3307` (MySQL), `6379` (Redis).

## Быстрый старт

```bash
# 1. Подготовить переменные окружения
cp .env.example .env

# 2. Поднять окружение (php-fpm, nginx, mysql, redis, воркер очереди)
docker compose up -d --build

# 3. Установить зависимости Composer
docker compose exec php composer install -d app

# 4. Применить миграции (создаст таблицы logs и files)
docker compose exec php ./app/yii migrate --interactive=0
```

После этого приложение доступно на <http://localhost:8080> (порт из `NGINX_PORT`).

Страница загрузки лог-файла: <http://localhost:8080/import>  
Страница со статистикой и графиками: <http://localhost:8080/stats>

## Переменные окружения (`.env`)

| Переменная         | По умолчанию    | Назначение                          |
|--------------------|-----------------|-------------------------------------|
| `NGINX_PORT`       | `8080`          | Порт nginx на хосте                 |
| `DB_PORT`          | `3307`          | Порт MySQL на хосте                 |
| `REDIS_PORT`       | `6379`          | Порт Redis на хосте                 |
| `DB_NAME`          | `nginx_parser`  | Имя базы данных                     |
| `DB_USER`          | `app`           | Пользователь БД                     |
| `DB_PASSWORD`      | `app`           | Пароль пользователя БД              |
| `DB_ROOT_PASSWORD` | `root`          | Пароль root MySQL                   |

## Как пользоваться

1. Откройте <http://localhost:8080/import>, выберите лог-файл nginx и загрузите его.
2. Файл сохраняется, считается его хэш и в очередь Redis (`nginx-logs:queue`)
   кладётся id задания. Если файл с таким хэшем уже импортирован - повторного импорта не будет.
3. Воркер импорта забирает задание, парсит строки и пишет их в БД батчами;
   прогресс пишется в Redis (`import:progress:{id}`).
4. На странице загрузки прогресс-бар обновляется по AJAX до 100% без перезагрузки.
5. Перейдите на <http://localhost:8080/stats> - выберите фильтры (даты, ОС, архитектура, боты), смотрите графики
   и таблицу; колонки таблицы сортируются по клику. Результаты агрегаций кэшируются в Redis на 5 минут.

## Консольные команды

Воркер запускается отдельной консольной коммандой:

```bash
# Воркер импорта: блокирующе ждёт задания из очереди Redis и обрабатывает их
docker compose exec php ./app/yii import/listen

# Параметры воркера:
#  --batchSize=512   размер батча вставки в БД
#  --timeout=5       таймаут ожидания задания (сек), 0 - ждать бесконечно
docker compose exec php ./app/yii import/listen --batchSize=1000 --timeout=0
```

Миграции:

```bash
docker compose exec php ./app/yii migrate
```

## Тесты

Unit-тесты покрывают классы парсинга строки лога (`NginxLogLineParser`) и User-Agent (`UserAgentInfoParser`):

```bash
# unit-тесты
docker compose exec php ./app/vendor/bin/phpunit -c ./app/phpunit.xml
```

## Архитектурные решения

- Очередь импорта реализована напрямую на Redis LIST (`lpush` / `brpop`), без брокера
  `yii2-queue`, чтобы соответствовать заданию.
- Парсеры (`NginxLogLineParser`, `UserAgentInfoParser`) - чистые классы без зависимостей
  от Yii, что упрощает их юнит-тестирование.
- Дедупликация: уникальный хэш файла в таблице `files`.
- Кэш агрегатов - `yii\redis\Cache`, ключ строится из нормализованных параметров фильтра, TTL 300 с.
